# Cross-Platform STL Validation Framework Guide

This guide explains how to use the validation framework to ensure the OpenSCAD generator produces STL files that match the web-based generator.

## Overview

The validation framework compares STL files generated by:
- **Reference**: Web-based generator (treated as ground truth)
- **Test**: OpenSCAD generator (this repository)

It validates:
- ✅ Geometric properties (volume, surface area, bounding box)
- ✅ Mesh quality (watertightness, face/vertex counts)
- ✅ Parameter schema consistency
- 🔧 Numeric surface deviation (optional, requires CloudCompare)

## Quick Start

### Prerequisites

1. **Install Required Tools**:
   ```bash
   # Python dependencies
   pip install -r tests/requirements.txt
   
   # OpenSCAD (required)
   # See tests/tool_versions.yml for installation instructions
   
   # CloudCompare (optional, for numeric deviation)
   # See tests/tool_versions.yml for installation instructions
   ```

2. **Verify Environment**:
   ```bash
   # Check OpenSCAD
   openscad --version
   
   # Check CloudCompare (optional)
   CloudCompare --version
   ```

### Step 1: Validate Parameter Schema

Before generating fixtures, ensure OpenSCAD parameters match the web UI:

```bash
python tests/validate_parameter_schema.py
```

This checks:
- All OpenSCAD parameters have web API mappings
- Default values match
- Types are compatible
- Enum values align

**Expected output**: All checks should pass (green).

### Step 2: Generate Reference Fixtures

Generate reference STL files from the web generator:

```bash
# Option 1: Generate all fixtures (requires web generator running)
python scripts/regenerate_fixtures.py --web-api-url http://localhost:5000

# Option 2: Generate a single test case
python scripts/regenerate_fixtures.py --test-case card_rounded_emboss_basic

# Option 3: Just check if web API is accessible
python scripts/regenerate_fixtures.py --check-only
```

**Note**: The web generator must be running. To start it:
```bash
# In the web generator repository
cd ../braille-card-and-cylinder-stl-generator
python app.py
```

### Step 3: Run Validation Tests

Run the cross-platform validation test suite:

```bash
# Run all tests
pytest tests/cross_platform_validation.py -v

# Run basic tests only (faster)
pytest tests/cross_platform_validation.py::TestBasicCases -v

# Run card tests only
pytest tests/cross_platform_validation.py::TestCardShapes -v

# Run cylinder tests only
pytest tests/cross_platform_validation.py::TestCylinderShapes -v

# Run a specific test case
pytest tests/cross_platform_validation.py -k "card_rounded_emboss_basic" -v

# Generate JSON report
pytest tests/cross_platform_validation.py --json-report --json-report-file=report.json
```

## Test Results

### Interpreting Results

Each test produces:

1. **Console output**: Real-time progress and summary
2. **JSON results**: Detailed metrics in `{test_name}_results.json`
3. **Test STL files**: Generated OpenSCAD STLs for inspection

### Comparison Metrics

| Metric | Tolerance | Required | Notes |
|--------|-----------|----------|-------|
| Volume | < 1.0% diff | ✅ Yes | Overall shape accuracy |
| Surface Area | < 0.5% diff | ✅ Yes | Detail preservation |
| Bounding Box | < 0.1 mm | ✅ Yes | Dimensional accuracy |
| Watertightness | Must match | ✅ Yes | Printability requirement |
| Face Count | Informational | ❌ No | Varies with resolution |
| Max Surface Deviation | < 0.05 mm | 🔧 Optional | Requires CloudCompare |

Tolerances are configured in `tests/compare_config.json`.

### Common Issues

**"OpenSCAD not found"**:
- Install OpenSCAD and ensure it's in PATH
- Or set `OPENSCAD_PATH` environment variable
- See `tests/tool_versions.yml` for installation help

**"Reference fixture not found"**:
- Run `python scripts/regenerate_fixtures.py` first
- Ensure the web generator is running

**"CloudCompare not found"**:
- This is optional - tests will skip numeric deviation checks
- Install CloudCompare for full validation (see `tests/tool_versions.yml`)

**Test failures**:
- Check `{test_name}_results.json` for detailed metrics
- Inspect the generated STL files visually
- Adjust tolerances in `tests/compare_config.json` if needed (after verification)

## Test Organization

### Test Cases

Test cases are defined in `tests/fixtures/cross_platform/test_cases.json`:

- **Basic tests** (`priority: high`): Core functionality, run these first
- **Custom tests** (`priority: medium`): Non-default parameters
- **Edge cases** (future): Boundary conditions

Each test case includes:
- Parameters (matching both web API and OpenSCAD)
- Expected properties (volume, area, etc.)
- Tags for filtering (card, cylinder, emboss, counter, etc.)

### Fixture Directory Structure

```
tests/fixtures/cross_platform/
├── test_cases.json              # Test case definitions
├── FIXTURES_VERSION.txt         # Version metadata (human-readable)
├── FIXTURES_VERSION.json        # Version metadata (machine-readable)
├── card_rounded_emboss_basic/   # Test case fixture
│   ├── reference.stl            # Reference STL from web generator
│   ├── params.json              # Parameters used
│   └── metadata.json            # Fixture metadata (checksums, etc.)
├── card_rounded_counter_basic/
│   └── ...
└── ...
```

## Running Tests in CI

### GitHub Actions Example

```yaml
name: STL Validation

on: [push, pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          lfs: true  # Pull LFS fixtures
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      
      - name: Install OpenSCAD
        run: |
          sudo add-apt-repository -y ppa:openscad/releases
          sudo apt-get update
          sudo apt-get install -y openscad
      
      - name: Install Python dependencies
        run: pip install -r tests/requirements.txt
      
      - name: Validate parameter schema
        run: python tests/validate_parameter_schema.py
      
      - name: Run validation tests
        run: pytest tests/cross_platform_validation.py -v --json-report --json-report-file=report.json
      
      - name: Upload results
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: validation-results
          path: |
            report.json
            *_results.json
```

## Advanced Usage

### Custom Tolerances

Edit `tests/compare_config.json` to adjust tolerances:

```json
{
  "tolerances": {
    "volume": {
      "percent": 1.0  // Adjust as needed
    },
    "surface_area": {
      "percent": 0.5
    }
  }
}
```

**Warning**: Only loosen tolerances after investigating failures. Tight tolerances catch real issues.

### Adding New Test Cases

1. Edit `tests/fixtures/cross_platform/test_cases.json`
2. Add new test case definition with parameters
3. Regenerate fixtures: `python scripts/regenerate_fixtures.py --test-case new_case_name`
4. Add test case name to parametrized test in `tests/cross_platform_validation.py`
5. Run: `pytest tests/cross_platform_validation.py -k "new_case_name"`

### Manual STL Generation

Generate STL with OpenSCAD directly:

```bash
# Using the runner script
python tests/openscad_runner.py \
  Braille_Card_And_Cylinder_STL_Generator.scad \
  output.stl \
  --params-json tests/fixtures/cross_platform/card_rounded_emboss_basic/params.json
```

### Manual Mesh Comparison

Compare two STL files directly:

```bash
python tests/mesh_comparison.py \
  reference.stl \
  test.stl \
  --config tests/compare_config.json \
  --output-json comparison.json
```

## Module Reference

### `tests/openscad_runner.py`
Automates OpenSCAD CLI execution. Handles parameter passing, timeouts, and error handling.

**Key classes**:
- `OpenSCADRunner`: Main runner class
- `OpenSCADResult`: Execution result with metadata

### `tests/mesh_comparison.py`
Compares mesh properties and (optionally) numeric surface deviation.

**Key classes**:
- `MeshComparator`: Main comparison class
- `MeshProperties`: Extracted mesh properties
- `ComparisonResult`: Comparison result with pass/fail

### `tests/conftest.py`
Pytest configuration and fixtures. Provides shared setup and utilities.

**Key fixtures**:
- `openscad_runner`: OpenSCAD runner instance
- `mesh_comparator`: Mesh comparator instance
- `test_cases`: Loaded test case definitions
- `fixtures_dir`: Fixture directory path

### `scripts/regenerate_fixtures.py`
Generates reference fixtures from web generator API.

**Usage**:
```bash
python scripts/regenerate_fixtures.py --web-api-url URL [--test-case NAME]
```

### `tests/validate_parameter_schema.py`
Validates OpenSCAD parameter definitions against mapping.

**Usage**:
```bash
python tests/validate_parameter_schema.py [--output-json results.json]
```

### `tests/cross_platform_validation.py`
Main pytest test suite with parametrized test cases.

**Test classes**:
- `TestCrossPlatformValidation`: All tests
- `TestBasicCases`: High-priority tests only
- `TestCardShapes`: Card tests only
- `TestCylinderShapes`: Cylinder tests only

## Configuration Files

### `tests/compare_config.json`
Tolerances, required checks, and tool settings.

### `tests/tool_versions.yml`
Tool version requirements and installation instructions.

### `tests/parameter_mapping.json`
Machine-readable parameter mapping (source of truth).

### `tests/fixtures/cross_platform/test_cases.json`
Test case definitions with parameters and expected properties.

## Troubleshooting

### Tests hang or time out
- Check OpenSCAD is rendering correctly
- Increase timeout in `tests/compare_config.json`
- Simplify test case parameters (lower quality settings)

### Large diffs in volume/area
- Verify both generators use same parameters
- Check for OpenSCAD syntax errors (missing semicolons, etc.)
- Inspect STL files visually in a viewer

### Fixtures out of date
- Regenerate: `python scripts/regenerate_fixtures.py`
- Check web generator version matches FIXTURES_VERSION.txt

### Git LFS issues
- Ensure Git LFS is installed: `git lfs install`
- Pull LFS files: `git lfs pull`
- Check `.gitattributes` includes `*.stl`
